# 操作系统结构（Chapter 2）思维导图及知识点细节

## 一、操作系统服务（Operating System Services）

### 1. 面向用户的服务

- **用户接口（User Interface）**
  - 命令行界面（CLI）：允许用户直接输入命令，如Unix/Linux的shell，部分命令为内置，部分为程序名，新增功能无需修改shell。
  - 图形用户界面（GUI）：采用桌面隐喻，通过鼠标、键盘操作，图标代表文件、程序等，如Windows、Mac OS X的Aqua界面。
  - 批处理界面（Batch）：用户提交作业序列，系统自动依次执行。
- **程序执行（Program Execution）**
  - 能将程序加载到内存并运行，支持正常或异常（出错）终止。
- **I/O操作（I/O Operations）**
  - 运行中的程序可能需要I/O，涉及文件或I/O设备。
- **文件系统操作（File-system Manipulation）**
  - 支持文件和目录的读写、创建、删除、搜索、信息列表、权限管理等。
- **通信（Communications）**
  - 进程间可交换信息，同一计算机或网络中的计算机均可。
  - 方式：共享内存或消息传递（由OS移动数据包）。
- **错误检测（Error Detection）**
  - 持续监控可能的错误，包括CPU、内存硬件、I/O设备、用户程序中的错误。
  - 对每种错误采取适当措施，确保计算正确一致，调试工具可提升使用效率。

### 2. 面向系统效率的服务

- **资源分配（Resource Allocation）**
  - 多用户或多作业并发时，为其分配资源，如CPU周期、主存、文件存储（有专门分配代码）、I/O设备（有通用请求和释放代码）。
- **记账（Accounting）**
  - 跟踪用户使用计算机资源的种类和数量。
- **保护和安全（Protection and Security）**
  - 保护：控制对系统资源的所有访问。
  - 安全：通过用户认证防止外部入侵，防御外部I/O设备的无效访问尝试，系统各部分需采取预防措施。

## 二、用户操作系统接口（User Operating System Interface）

### 1. 命令行界面（CLI）

- 命令解释器允许直接输入命令，可在内核中实现或由系统程序实现，有多种版本（shell）。
- 主要功能：获取用户命令并执行，命令可内置或为程序名，后者新增功能无需修改shell。

### 2. 图形用户界面（GUI）

- 友好的桌面隐喻界面，通常使用鼠标、键盘和显示器。
- 图标代表文件、程序、操作等，鼠标在界面对象上的不同操作触发不同动作。
- 由Xerox PARC发明，许多系统同时包含CLI和GUI，如Windows（GUI+“命令”shell）、Mac OS X（Aqua GUI+UNIX内核及shell）、Unix和Linux（CLI+可选GUI如CDE、KDE、GNOME）。

### 3. 触摸屏界面（Touchscreen Interfaces）

- 适用于触摸屏设备，无需鼠标，基于手势进行操作和选择，通过虚拟键盘输入文本。

## 三、系统调用（System Calls）

### 1. 基本概念

- 是操作系统提供服务的编程接口，通常用高级语言（如C、C++）编写。
- 程序多通过高级应用程序接口（API）访问，而非直接使用系统调用。
- 常见API：Windows的Win32 API、基于POSIX的系统（如UNIX、Linux、Mac OS X）的POSIX API、Java虚拟机的Java API。

### 2. 示例

- **文件复制的系统调用序列**：获取输入/输出文件名、打开输入文件（不存在则中止）、创建输出文件（存在则中止）、循环读写、关闭输出文件、输出完成信息、正常终止。
- **标准API示例（UNIX/Linux的read()）**：包含头文件unistd.h，参数为文件描述符、数据缓冲区、最大读取字节数，成功返回读取字节数，0表示文件结束，错误返回-1。

### 3. 实现

- 每个系统调用关联一个编号，系统调用接口维护按编号索引的表。
- 接口调用OS内核中的目标系统调用，返回状态和返回值，调用者无需了解实现细节，只需遵循API。
- 由运行时支持库（编译器包含的库函数集）管理，隐藏OS接口细节。

### 4. 参数传递方式

- 寄存器传递：简单，但参数可能超过寄存器数量。
- 内存块/表传递：参数存储在内存块或表中，将块地址作为参数放在寄存器中，如Linux和Solaris。
- 栈传递：程序将参数压入栈，OS从栈中弹出，无参数数量和长度限制。

### 5. 系统调用类型

- **进程控制**：结束、中止、加载、执行、创建进程、终止进程、获取/设置进程属性、等待时间/事件、信号事件、内存分配释放、错误时内存转储、调试器、共享数据访问锁等。
- **文件管理**：创建/删除文件、打开/关闭文件、读写、重定位、获取/设置文件属性。
- **设备管理**：请求/释放设备、读写、重定位、获取/设置设备属性、逻辑上附加/分离设备。
- **信息维护**：获取/设置时间日期、系统数据、进程/文件/设备属性。
- **通信**：创建/删除通信连接、消息发送接收（消息传递模型）、创建和访问内存区域（共享内存模型）、传递状态信息、附加/分离远程设备。
- **保护**：控制资源访问、获取/设置权限、允许/拒绝用户访问。

### 6. Windows和Unix系统调用示例

| 类型     | Windows系统调用                  | Unix系统调用              |
| -------- | -------------------------------- | ------------------------- |
| 进程控制 | ExitProcess()、CreateProcess()等 | fork()、exit()、wait()等  |
| 文件操作 | CreateFile()、ReadFile()等       | open()、read()、close()等 |
| 设备操作 | SetConsoleMode()等               | ioctl()等                 |
| 信息维护 | GetCurrentProcessID()等          | getpid()、alarm()等       |
| 通信     | CreatePipe()等                   | pipe()、mmap()等          |
| 保护     | SetFileSecurity()等              | chmod()、chown()等        |

## 四、系统程序（System Programs）

### 1. 功能

- 提供程序开发和执行的便捷环境，部分是系统调用的用户接口，部分更复杂。

### 2. 分类

- **文件操作（File Manipulation）**：创建、删除、复制、重命名、打印、转储、列表等文件和目录操作。
- **状态信息（Status Information）**：向系统请求信息（日期、时间、内存、磁盘空间、用户数等），提供性能、日志、调试信息，格式化输出到终端等，部分系统用注册表存储配置信息。
- **文件修改（File Modification）**：文本编辑器创建修改文件，特殊命令搜索文件内容或转换文本。
- **编程语言支持（Programming-language Support）**：提供编译器、汇编器、调试器、解释器。
- **程序加载和执行（Program Loading and Execution）**：绝对加载器、可重定位加载器、链接编辑器、覆盖加载器、高级语言和机器语言调试系统。
- **通信（Communications）**：创建进程、用户、计算机系统间的虚拟连接，支持消息发送、网页浏览、电子邮件、远程登录、文件传输。
- **后台服务（Background Services）**：启动于 boot 时，部分用于系统启动后终止，部分从启动到关闭运行，提供磁盘检查、进程调度、错误日志、打印等功能，在用户态运行，称为服务、子系统、守护进程。
- **应用程序（Application Programs）**：与系统无关，由用户运行，通常不视为OS部分，通过命令行、鼠标点击、手指触碰启动。

## 五、操作系统设计与实现（Operating System Design and Implementation）

### 1. 设计目标

- **用户目标**：方便使用、易于学习、可靠、安全、快速。
- **系统目标**：易于设计、实现、维护，灵活、可靠、无错、高效。

### 2. 重要原则

- 分离策略（What will be done?）和机制（How to do it?），便于后续修改策略，保持灵活性。

### 3. 实现语言

- 早期用汇编语言，后用Algol、PL/1，现在主要用C、C++，通常混合使用：底层用汇编，主体用C，系统程序用C、C++、脚本语言（如PERL、Python、shell脚本）。
- 高级语言更易移植到其他硬件，但速度较慢，仿真可使OS在非原生硬件上运行。

## 六、操作系统结构（Operating System Structure）

### 1. 简单结构（Simple Structure）

- 如MS-DOS，为在有限空间提供最多功能，未模块化，接口和功能层次未良好分离，包含应用程序、驻留系统程序、MS-DOS设备驱动程序、ROM BIOS设备驱动程序。

### 2. UNIX结构

- 受硬件功能限制，结构有限，由系统程序和内核组成。
- 内核：位于系统调用接口之下，物理硬件之上，提供文件系统、CPU调度、内存管理等OS功能。

### 3. 分层方法（Layered Approach）

- OS分为多个层，底层（0层）为硬件，最高层（N层）为用户接口，每层仅使用下层的功能和服务，实现模块化。

### 4. 微内核结构（Microkernel System Structure）

- 将内核中大部分功能移到用户空间，通过消息传递通信，如Mach，Mac OS X内核部分基于此。
- 优点：易于扩展、易移植到新架构、更可靠（内核态代码少）、更安全。
- 缺点：用户态与内核态通信的性能开销。
- 组成：用户态的应用程序、设备驱动、文件系统等，内核态的微内核（进程管理、内存管理、CPU调度、进程间通信）。

### 5. 模块（Modules）

- 现代OS多实现可加载内核模块，采用面向对象方法，各核心组件独立，通过已知接口通信，按需加载到内核，比分层更灵活，如Linux、Solaris。

### 6. 混合系统（Hybrid Systems）

- 多数现代OS并非纯模型，结合多种方法满足性能、安全、可用性需求。
- 如Linux和Solaris：内核地址空间为单块式，加模块动态加载功能；Windows：主要单块式，加微内核支持不同子系统；Mac OS X：分层，Aqua UI加Cocoa编程环境，内核包含Mach微内核、BSD Unix部分、I/O工具包和可动态加载模块。

## 七、操作系统调试（Operating System Debugging）

### 1. 调试概念

- 调试是发现和修复错误（bugs），OS生成包含错误信息的日志文件。
- 应用程序失败生成核心转储文件（捕获进程内存），OS失败生成崩溃转储文件（捕获内核内存）。

### 2. 性能调优（Performance Tuning）

- 优化系统性能，去除瓶颈，OS需提供计算和显示系统行为度量的方法，如“top”程序、Windows任务管理器。
- DTrace工具（Solaris、FreeBSD、Mac OS X）：可在生产系统上实时检测，探针在提供者执行代码时触发，捕获状态数据发送给消费者。

## 八、操作系统生成（Operating System Generation）

- OS设计为可在一类机器上运行，需为每个特定计算机站点配置。
- SYSGEN程序获取硬件系统的具体配置信息，用于构建特定系统的编译内核或系统调优，比通用内核更高效。

## 九、系统启动（System Boot）

- 系统加电后，从固定内存位置开始执行，固件ROM存放初始引导代码。
- 引导程序（bootstrap loader）：小代码，存储在ROM或EEPROM中，定位内核，加载到内存并启动，有时分两步（ROM代码加载固定位置的引导块，再加载引导程序）。
- 常见引导程序如GRUB，允许从多个磁盘、版本中选择内核及设置内核选项，内核加载后系统开始运行。
