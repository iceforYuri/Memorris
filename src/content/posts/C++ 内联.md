# 基础原理


我们都知道，函数调用在程序执行时会带来一定的开销：

1. **参数压栈** ：将实参值压入栈中。
2. **跳转** ：将程序执行流跳转到函数体的起始地址。
3. **寄存器保存** ：保存调用方现场的寄存器状态。
4. **栈帧创建** ：为被调函数创建新的栈帧。
5. **返回** ：执行完毕后，恢复现场，跳转回调用方。

这些步骤虽然单个来看很快，但如果一个函数被频繁调用（例如，在一个紧密的循环中），这些开销就会累积起来，成为性能的瓶颈。

这是一种速度的诱惑，让我们尝试将函数内容替换函数调用位置

那么**核心思想：用函数的代码体直接替换函数调用点**

# 内联关键字 `inline` 的本质

内联的声明是根据函数规模大小，向编译器建议直接把它的代码内容**复制**到调用它的地方

真正核心：在于消除函数调用的过程
